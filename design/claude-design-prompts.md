# Claude Design 요청 모음 — 약국 어디가

각 화면을 디자인할 때 **① 공통 스타일 블록을 먼저 붙이고 → ② 화면 프롬프트**를 이어 붙인다.
가능하면 **기존과 같은 Claude Design 프로젝트 안에서** 작업해 스타일을 재사용할 것.
(이미 만든 메인/상세 화면을 참고(reference)로 첨부하면 더 일관됨)

---

## ① 공통 스타일 블록 (매 프롬프트 맨 위에 붙이기)

```
'약국 어디가' — 위치 기반 약국 찾기 웹앱. 기존 화면들과 같은 디자인 시스템 유지.
[컬러] Primary #0B8FAC / Deep #086B82 / Accent #22D3EE / 지도 #0C2A33
       상태색 운영중 #16A34A / 곧마감 #EA9006 / 종료 #94A3B8
       Ink #0E2A33 / Muted #5E7C88 / Line #E4EDF1 / 배경 #E8EFF2
[타이포] 한글 시스템 스택(Pretendard 느낌), 숫자 tabular
[라운드] 버튼11 / 카드15 / 시트26 / 칩·뱃지 full
[톤] 깔끔하고 신뢰감 있는 의료 서비스. 모바일 우선.
```

---

## ② 화면별 프롬프트

### 위치 권한 요청 (최초 진입)
```
앱 첫 진입 시 위치 사용 동의를 받는 화면. 중앙에 위치 핀 일러스트(accent 시안
글로우), "내 주변 약국을 찾으려면 위치 권한이 필요해요" 안내, [위치 허용] 기본
버튼 + [지역 직접 선택] 보조 텍스트 버튼. 부드러운 진입 애니메이션.
```

### 위치 권한 거부 폴백
```
사용자가 위치 권한을 거부했을 때의 화면. "위치 없이도 지역을 직접 골라
약국을 찾을 수 있어요" 안내, 지역 선택 입력(예: 강남구 역삼동), [다시 위치 허용]
링크. 막다른 느낌 대신 대안을 주는 톤.
```

### 빈 결과
```
주변에 운영 중인 약국이 없을 때의 빈 상태. 차분한 일러스트 + "현재 주변에 운영
중인 약국이 없어요" + [검색 반경 넓히기] / [지역 변경] 버튼. 절망적이지 않게.
```

### 에러 / 재시도
```
약국 정보를 불러오지 못했을 때(네트워크·서버 오류). "약국 정보를 불러오지
못했어요" + 원인 한 줄 + [다시 시도] 버튼. 사과체 말고 명확하게.
```

### 검색 (오버레이)
```
지역·약국 이름 검색 오버레이. 상단 검색 입력(자동 포커스) + 최근 검색어 칩 +
입력 시 실시간 결과 리스트(약국 카드 축약형: 이름·운영상태·거리). 메인 위로
부드럽게 슬라이드 인.
```

### 필터 시트 (바텀시트)
```
약국 필터 바텀시트. 토글: 운영 중만 / 심야 운영 / 공휴일 운영 / 연중무휴.
거리 범위 슬라이더(1~5km). 하단 [적용] 버튼 + [초기화]. 모바일 바텀시트 형태.
```

### 즐겨찾기 목록
```
저장한 약국 목록 화면. 상단 "즐겨찾기" 타이틀, 약국 카드 리스트(메인과 동일
카드), 각 카드에 채워진 별 아이콘. 비어 있을 때는 "자주 가는 약국을 저장해
보세요" 빈 상태.
```

### 설정 / 정보
```
설정·정보 화면. 항목 리스트: 데이터 출처(식약처 e약은요), 앱 정보/버전,
면책 고지("본 서비스는 약국 정보 제공 보조 목적이며 의료·복약 상담을
대체하지 않습니다"). 깔끔한 설정 리스트 UI.
```

## 참고
- 이미 디자인된 것: 메인(지도+목록), 약국 상세 — 모바일/데스크탑
- 로딩(레이더 스캔)은 메인 진입 애니메이션에 포함됨
- 데스크탑 변형이 필요한 화면은 끝에 "왼쪽 패널 + 오른쪽 큰 지도 데스크탑 버전도" 추가
