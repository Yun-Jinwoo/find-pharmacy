# 약국 어디가 — 프로젝트 인수인계 (HANDOFF)

> 이 파일은 로컬 Claude Code가 작업을 이어받기 위한 맥락 정리본이다. 먼저 이 파일을 읽고 시작할 것.

## 1. 프로젝트 개요

- **이름**: 약국 어디가
- **한 줄**: 사용자 위치 기반으로 **지금 운영 중인 약국**을 지도+거리순 목록으로 보여주는 웹 서비스 (휴일·야간 특화)
- **형태**: 반응형 **웹** (React Native 아님. 모바일은 모바일 브라우저 대응)
- **개발자**: 1인. 프론트엔드(React/Next.js) 경험 있음, 백엔드 처음.
- **포지셔닝**: 약사 대체 아님 → 상비약 찾기 보조 + 포트폴리오. 차별점은 **모바일 UX + 직관적 지도**(데이터 정확도가 아님 — 공공데이터 한계 동일).

## 2. 확정된 기술 스택

| 영역 | 선택 | 이유 |
| --- | --- | --- |
| 프레임워크 | **Next.js (App Router)** | 개발자가 아는 기술 + 백엔드 내장 |
| 백엔드 | **Next.js Route Handlers** | 별도 서버 불필요(FastAPI 등 학습 X). 공공 API 프록시 + 키 숨김 + "현재 운영중" 계산 |
| 배포 | **Vercel** | 무료, HTTPS 자동(위치 권한 필수 조건) |
| 지도 | **카카오맵 JS SDK** | 한국 주소·길찾기 최적 |
| DB | **없음 (MVP)** | 즐겨찾기는 localStorage |
| 데이터 | **공공데이터포털 약국 정보 API** (좌표·요일별 영업시간) | ⚠️ 검증 필요(아래) |

## 3. MVP 범위

위치 기반 약국 검색 / 지도 뷰(마커) / 거리순 목록 / 약국 상세(주소·시간·전화) / 길찾기 연동.
확장(후순위): 필터, 즐겨찾기, 약품 재고검색(공개 API 사실상 없음 → 보류).

## 5. ⚠️ 가장 중요한 리스크 — 먼저 검증할 것

- 약국 **운영시간 데이터 정확도**가 핵심 리스크. 공공데이터/휴일지킴이/지도앱이 사실상 같은 원천이라 정확도 천장이 같음. 임시휴무·공휴일 영업은 자주 부정확.
- **"현재 운영중" 판정 로직**(요일·시각 + 공휴일 판단)이 유일하게 까다로운 부분.
- → **권장: 코드 짜기 전에 공공데이터 약국 API를 실제로 받아 영업시간·좌표 필드가 쓸 만한지 먼저 검증.**

## 6. 권장 진행 순서

1. **공공데이터 약국 API 검증** (영업시간·좌표 확인) ← 성패의 핵심, 먼저
2. **Next.js 셋업** + `design/`의 디자인 시스템을 React 컴포넌트(+Tailwind)로 이식
3. **카카오맵 + geolocation + 거리(Haversine) 정렬** 연결 → 실제 약국 표시
4. 상세/길찾기, 그다음 확장(필터·즐겨찾기)

## 7. 지금 할 일 (이 세션)

- Claude Design `약국 어디가 데스크탑.dc.html`을 가져와 **데스크탑 화면을 Next.js로 구현**.
- 단, 위 5번(데이터 검증)을 아직 안 했다면, 화면은 **mock 데이터**로 먼저 만들고 API 연결은 검증 후에.
