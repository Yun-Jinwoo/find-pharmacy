# 접근성(a11y) 개선 — 키보드 내비게이션 & 색 대비

> 클릭 전용 UI를 키보드로도 쓸 수 있게 만들고, WCAG AA 색 대비 미달을 해소한 기록.
> 관련 커밋: `4dd7345`(키보드·aria), `69e7b29`(DevNav 제거), `9cec202`(색 대비)

## 1. 진단 — Lighthouse + 코드 감사

Lighthouse 접근성 감사에서 색 대비 미달이 잡혔고, 코드를 보니 더 근본적인
문제가 있었다: **약국 카드가 `<div onClick>`** 이라 마우스로만 조작 가능하고
키보드(Tab/Enter) 사용자는 목록을 아예 쓸 수 없었다.

## 2. 키보드 내비게이션 & 시맨틱 (커밋 4dd7345)

- 약국 카드(데스크탑/모바일)와 검색 결과 행을 `role="button"` +
  `tabIndex={0}` + `onKeyDown`(Enter/Space)으로 **키보드 조작 가능**하게 함.
- 카드에 `aria-label`(이름·운영상태·"상세 정보 보기"), 즐겨찾기 아이콘
  버튼에 상태별 `aria-label`(추가/해제) 추가.
- `globals.css`에 전역 `:focus-visible` 아웃라인 — 키보드 탐색 시에만
  모든 포커스 요소에 표시(마우스/평상시엔 없음). 처음엔 Tailwind
  `focus-visible:ring-2`로 했으나 유틸리티가 생성되지 않아, 전역 CSS
  규칙으로 바꿔 모든 포커스 요소를 한 번에 커버.

**검증**: 카드에 Tab 포커스 → Enter로 상세 패널 오픈 확인.

## 3. DevNav 제거 (커밋 69e7b29)

개발 중 화면 전환용 임시 패널(DevNav)이 **환경 가드 없이 프로덕션에도
렌더**되고 있었다. 실사용자에게 노출될뿐더러 색 대비도 미달이었다.
컴포넌트·사용처·전용 핸들러(switchState)를 모두 제거.

## 4. 색 대비 — WCAG AA (커밋 9cec202)

흰 배경/흰 글씨 기준 4.5:1 미달 색 2종을 조정. 변경 전 디자인 토큰
비교안을 시각적으로 확인(Artifact)한 뒤 반영.

| 토큰 | 변경 | 대비 | 쓰임 |
| --- | --- | --- | --- |
| `--primary` | #0B8FAC → **#097A96** | 3.79 → 4.96 | 버튼 흰 글씨, 강조 텍스트("N곳") |
| `--muted` | #5E7C88 → **#587582** | 4.45 → 4.90 | 본문 보조 텍스트 |

- primary를 토큰 미사용으로 하드코딩한 버튼(에러/필터/위치권한/지역선택
  화면)도 함께 반영.
- 큰 제목·아이콘·테두리 인상은 거의 유지.

**검증(프로덕션 빌드 + Lighthouse)**: 접근성 **95 → 100**,
color-contrast 감사 **PASS**(실패 0).

## 5. 판단력을 보여준 지점

- **감사의 성격을 이해하고 우선순위를 나눴다**: `--muted`는 4.45로 사람
  눈엔 구분 안 되는 "감사 통과용", `--primary`는 3.79로 저시력·야외에서
  실제로 안 읽히는 "진짜 문제"임을 구분해 설명. 다만 Lighthouse
  color-contrast는 **all-or-nothing**(하나만 실패해도 감사 전체 FAIL)이라,
  점수를 올리려면 둘 다 고쳐야 한다는 점도 근거로 제시.
- **측정 함정 발견**: 이전 세션의 `next start`가 포트에 살아있어 새 빌드가
  아닌 옛 빌드를 서빙 → Lighthouse가 스테일 결과를 냈다. 리스너를 정리하고
  재측정해 실제 반영을 확인.

## 6. 남은 것

- 랜딩 페이지(`app/LandingClient.tsx`, `app/landing.css`)는 브랜드 색을 자체 정의·하드코딩해서, 메인
  앱만 바꾼 지금 두 화면의 청록/텍스트 색이 미세하게 다르다. 통일 필요
  (별도 작업으로 분리).

## 7. 한 줄 요약

> `<div onClick>` 카드를 키보드로 조작 가능하게 만들고(role/tabIndex/keydown +
> 전역 focus-visible), 프로덕션에 새던 개발 패널을 제거하고, WCAG AA 색 대비
> 미달을 토큰 조정으로 해소해 Lighthouse 접근성 100을 달성했다.
